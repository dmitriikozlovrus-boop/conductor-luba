# System Issue

## Описание

System Issue — это ошибка, сбой, ограничение, недостаток или нежелательное поведение ИИ-системы, ее компонентов, правил, данных, интеграций или процессов.

System Issue фиксирует проблему самой системы, а не проблему пользователя, бизнеса или проекта.

---

## Основная цель

Зафиксировать проблему системы для последующего анализа, исправления и улучшения качества работы.

---

## Относится

- ошибки классификации;
- ошибки маршрутизации;
- ошибки заполнения данных;
- ошибки интеграций;
- ошибки синхронизации;
- архитектурные недостатки;
- неэффективные инструкции;
- избыточный расход токенов;
- ошибки агентов;
- ошибки автоматизаций;
- ошибки памяти;
- ошибки обработки данных;
- проблемы качества данных;
- проблемы производительности;
- проблемы пользовательского опыта.

---

## Не относится

- жизненные проблемы пользователя;
- бизнес-проблемы;
- рабочие проблемы;
- исследовательские вопросы;
- идеи;
- задачи;
- события календаря;
- коммуникации;
- контакты;
- компании.

---

## Связанные сущности

- Problem;
- Task;
- Study;
- Project;
- Stream;
- Improvement.

---

## Ключевые признаки

- проблема существует внутри системы;
- проблема ухудшает качество работы системы;
- проблема может требовать изменения кода, архитектуры, инструкций, данных или процессов;
- проблема может повторяться;
- устранение проблемы приводит к улучшению системы.
