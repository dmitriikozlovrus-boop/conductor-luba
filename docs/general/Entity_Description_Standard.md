# Entity Description Standard

## Назначение

Описание сущности должно позволять ИИ:

- определять принадлежность новой информации;
- использовать правильный контекст;
- различать похожие сущности;
- принимать решения о классификации и маршрутизации информации;
- определять, когда необходимо создать сущность;
- определять, когда создавать другую сущность вместо данной.

# Требования к качеству

Описание должно:

- четко определять границы сущности;
- содержать конкретные примеры;
- минимизировать пересечения с другими сущностями;
- позволять классифицировать информацию без участия человека;
- содержать правила создания сущности;
- содержать правила отказа от создания сущности;
- обеспечивать однозначную маршрутизацию информации.

---

# Контрольные вопросы

1. Что представляет собой сущность?
2. Какова её основная цель?
3. Когда сущность должна создаваться?
4. Когда сущность не должна создаваться?
5. Что относится к сущности?
6. Что не относится к сущности?
7. С какими сущностями она обычно связана?
8. По каким признакам её можно распознать?
9. Чем она отличается от соседних сущностей?

Если хотя бы на один вопрос невозможно ответить однозначно, описание требует доработки.

---

# Главный принцип

Описание сущности должно быть достаточным для того, чтобы ИИ мог самостоятельно:

1. классифицировать входящую информацию;
2. выбрать правильную сущность;
3. отказаться от неправильной сущности;
4. определить связи между сущностями;
5. маршрутизировать информацию без участия человека.
