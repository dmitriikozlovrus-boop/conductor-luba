# Task

## Описание

Task — это конкретное действие, обязательство или следующий шаг, который можно выполнить и завершить.

Task создается только тогда, когда входящая информация требует действия, а не фиксации проблемы, идеи, знания, коммуникации или вопроса для изучения.

---

## Основная цель

Зафиксировать действие, которое необходимо выполнить.

---

## Когда создавать

Создавать Task, если:

- есть конкретное действие;
- есть обязательство или поручение;
- есть следующий шаг по проекту;
- есть следующий шаг по проблеме;
- есть следующий шаг по исследованию;
- есть следующий шаг по коммуникации;
- действие можно завершить и отметить как выполненное.

---

## Когда НЕ создавать

- если нужно зафиксировать проблему, риск или ограничение → Problem;
- если нужно изучить вопрос, собрать информацию или провести анализ → Study;
- если появилась идея, гипотеза или возможность → Idea;
- если нужно зарезервировать время → Event;
- если фиксируется человек → Contact;
- если фиксируется организация → Company;
- если фиксируется взаимодействие → Communication;
- если отсутствует конкретное действие → не создавать Task.

---

## Относится

- поручения;
- обязательства;
- следующие шаги;
- звонки;
- письма;
- сообщения;
- проверки;
- согласования;
- обновления;
- организационные действия;
- действия после встреч;
- действия после исследований;
- действия после выявления проблем.

---

## Не относится

- проблемы;
- исследования;
- идеи;
- заметки без действия;
- факты без действия;
- контакты;
- компании;
- коммуникации;
- знания;
- события календаря без самостоятельного действия.

---

## Связанные сущности

- Project;
- Stream;
- Problem;
- Study;
- Event;
- Contact;
- Company;
- Communication;
- Topic.

---

## Ключевые признаки

- есть действие;
- есть ожидаемый результат;
- действие можно завершить;
- после выполнения состояние изменится;
- существует исполнитель.
